# Code of Conduct

## Our standard

This project aims to be professional, direct, and respectful.

Contributors are expected to:

- focus on the technical problem,
- explain tradeoffs clearly,
- avoid personal attacks,
- keep feedback actionable,
- respect users working in multilingual and cross-platform environments.

## Unacceptable behavior

- harassment or discrimination,
- hostile or insulting language,
- bad-faith review behavior,
- repeated low-signal noise that blocks productive collaboration,
- sharing private or sensitive project information without permission.

## Enforcement

Project maintainers may edit, remove, or reject contributions and discussions that violate this standard.

## Scope

This applies to issues, pull requests, discussions, and other project communication channels.
